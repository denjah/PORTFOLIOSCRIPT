# ТЕХНИЧЕСКАЯ СПЕЦИФИКАЦИЯ ПРОЕКТА

## Общие параметры макета

### Размеры и структура
- **Ширина макета**: 1440px (фиксированная)
- **Высота макета**: динамическая (зависит от контента)
- **Колонки**: 4 колонки с равномерным распределением
- **Боковые отступы**: 40px слева и справа (для контента, не для фона)
- **Отступ между группами карточек**: 100px (опционально, может быть бесшовное соединение)

### Фон
- **Формат**: два изображения (версия А и версия Б)
- **Минимальная высота**: 3000px (рекомендуется)
- **Поведение**: скроллится вместе с контентом, автоматическое чередование А → Б → А → Б...
- **Файлы**: 
  - `project-name-bg.jpg` - версия А (оригинальная)
  - `project-name-bg-B.jpg` - версия Б (вертикально зеркальная)
- **Подгрузка**: новые сегменты автоматически добавляются по мере скролла до конца контента
- **Уникальность**: отдельные файлы фона для каждого проекта

### Пропорции карточек товара
- **Соотношение сторон**: 3:4 (portrait) или 1:1 (square) - переключаемо
- **Применение**: все карточки товара в одном макете должны быть одного формата
- **Переключение**: через панель управления в правом верхнем углу или консольные команды
- **По умолчанию**: 3:4 (portrait)

## Структура макета (4 раздела)

### Раздел 1: Вступление и приветствие
**Назначение**: Краткое введение с обзорной графикой проекта

**Компоненты**:
- Приветственный текст (до 300 символов, редактируемый)
- Общая ознакомительная графика из всего проекта
- Декоративные иконки (2-3 шт на каждые 900px высоты)

**Текст по умолчанию**:
```
Представляю вашему вниманию серию карточек товара, разработанную для электро-коробов BT-AT и BT-AG. Этот проект охватил полный цикл создания маркетинговых материалов – от профессиональной фотосъемки до финальной верстки и подготовки файлов для различных каналов.
```

**Назначение**: Детальный показ начала и хода процесса работы

**Компоненты**:
- Визуализация этапов работы
- Текстовые блоки с описанием процессов
- Декоративные иконки

**Текстовые блоки**:

2. **Контент и тексты**: "Сбор и анализ информации о продукте, разработка информативного и лаконичного текста с акцентом на ключевых преимуществах и характеристиках товара."

3. **Фотография и обработка**: "Предметная фотосъемка продукции с акцентом на детали и текстуру, последующая профессиональная ретушь, обтравка и цветокоррекция для создания привлекательных и реалистичных изображений."

---

### Раздел 3: Галерея карточек товара
**Назначение**: Основной контент — демонстрация готовых карточек

**Система раскладки**:
- Карточки организованы в группы по 8-15 штук
- Первая карточка каждой группы занимает **2 колонки** (двойная ширина)
- Остальные карточки занимают **1 колонку**
- Между группами отступ 100px или бесшовное продолжение

**Сетка**:
- Базовая сетка: 4 колонки
- CSS Grid с автоматическим размещением
- Возможность ручного перемещения карточек (drag-and-drop или swap функция)

**КРИТИЧЕСКИ ВАЖНО для автоматизации**:
```css
.gallery-grid {
  display: grid;
  grid-template-columns: repeat(4, 1fr);
  gap: 20px;
  padding: 0 40px;
}

.card {
  aspect-ratio: 3 / 4; /* По умолчанию portrait */
  grid-column: span 1;
}

.card--portrait {
  aspect-ratio: 3 / 4; /* Вертикальный формат */
}

.card--square {
  aspect-ratio: 1 / 1; /* Квадратный формат */
}

.card--first {
  grid-column: span 2; /* Первая карточка группы */
}
```

## Система автоматизации

### Требования к автоматизации

1. **Автоматическая раскладка карточек**
   - Карточки автоматически размещаются по сетке 4 колонки
   - Первая карточка группы автоматически получает класс `.card--first`
   - Соблюдение пропорций 3:4 через `aspect-ratio`

2. **Ручная корректировка**
   - Возможность swap (обмена местами) любых двух карточек
   - Возможность переназначить, какая карточка будет "первой" в группе
   - Drag-and-drop интерфейс (опционально)

3. **Управление группами**
   - Определение начала новой группы (data-атрибут или класс)
   - Автоматический расчет отступов между группами
   - Возможность бесшовного соединения групп

4. **Экспорт**
   - Генерация PDF с сохранением качества
   - Разрезка на отдельные файлы 1440×900px для Behance
   - Бесшовная стыковка соседних секций

## Правила кода

### Структура файлов
- `index.html` — основной файл разметки
- `styles.css` — стили (или отдельные файлы для каждого раздела)
- `script.js` — логика автоматизации и интерактивности
- `assets/backgrounds/` — фоновые изображения проектов
- `assets/cards/` — карточки товаров (автоматическая загрузка)
- `assets/icons/` — декоративные иконки

### CSS Guidelines
- Используй CSS Grid для основной сетки
- `aspect-ratio` для карточек товара (поддержка современных браузеров)
- Responsive не требуется (фиксированная ширина 1440px)
- BEM методология для именования классов
- CSS Custom Properties для переменных (цвета, размеры, отступы)

### JavaScript Guidelines
- Используй `const` вместо `function`
- Префикс `handle` для event handlers: `handleCardSwap`, `handleGroupSplit`
- Descriptive naming: `cardElements`, `galleryGrid`, `firstCardInGroup`
- Early returns для читаемости
- Модульная структура: отдельные функции для каждой задачи

### HTML Guidelines
- Semantic HTML5: `<section>`, `<article>`, `<figure>`, `<header>`, `<footer>`
- Data-атрибуты для управления: `data-group`, `data-card-index`, `data-first`
- Accessibility: alt-тексты, ARIA-labels где необходимо

## Рабочий процесс

### Этап 1: Создание шаблона
1. Создать базовую HTML структуру с 4 разделами
2. Настроить CSS Grid систему
3. Имплементировать систему фоновых изображений
4. Разместить декоративные иконки

### Этап 2: Автоматизация
1. Создать функцию автозагрузки карточек из папки
2. Автоматическое применение класса `.card--first`
3. Функционал swap карточек
4. Группировка карточек (8-15 штук)

### Этап 3: Наполнение контентом
1. Заменить placeholder-тексты на реальные
2. Загрузить карточки товара
3. Загрузить фоновое изображение проекта
4. Скорректировать расположение иконок

### Этап 4: Экспорт
1. Проверка финального макета
2. Генерация PDF (через browser print или библиотеки)
3. Разрезка на секции 1440×900px
4. Загрузка на Behance

## Экспорт и финализация

- Размер: ширина 1440px, высота динамическая

- Высота секции: 900px

## Технические примечания

### Browser Support
- Современные браузеры (Chrome, Firefox, Safari, Edge)
- CSS Grid и aspect-ratio обязательны
- JavaScript ES6+

### Performance
- Lazy loading для изображений не требуется (PDF export)
- Optimize images: WebP для веб-превью, оригиналы для PDF
- Background: один файл, multiple use (repeat-y)

### Debugging
- Console.log для отладки группировки
- Визуальные маркеры для проверки сетки (dev mode)
- Проверка aspect-ratio через DevTools

## Итоговая философия

**Автоматизация без потери контроля**: Система должна автоматически делать рутинные операции (раскладка по сетке, пропорции), но давать полный контроль для творческих решений (перестановка, группировка, акценты).

**Шаблон как конструктор**: Один раз создаем систему, затем быстро наполняем контентом для каждого нового проекта в портфолио.

**Качество превыше скорости**: Код должен быть чистым и понятным для будущих модификаций, даже если это займет больше времени на первоначальную разработку.
